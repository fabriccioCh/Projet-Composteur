projet(composteur connecté) : 
-Programmation Arduino : Lecture des données des capteurs(température, humidité et j'ai rajouter pression hpa) , Transmission des données vers le broker MQTT via NB-Iot et Activation de la pompe selon les seuils définis.
-Broker MQTT : Centralisation des messages provenant de l'Arduino , Gestion des topics , Publication des données pour les autres services(API, App mobile, etc.) 
-Sécurisation : Chiffrement TLS/SSL - protection des messages MQTT échangés entre les clients et le broker , Authentification des clients - Implémentation de noms d'utilisateur et mots de passe pour chaque appareil connecté, Autorisations par sujets : Définition de permission spécifiques pour chaque topic afin de limiter les accès non autorisés, Chiffrement des données - Envoi des données des capteur via des protocoles sécurisés(dans ce ca la SSL).
 -API : Interface entre le broker MQTT, la base de données InfluxDB, et les utilisateurs, Gère les requêtes - récuperation des données (GET) et envoi de commandes(POST), Sécurisée avec JWT pour l'authentification et la gestion des accès.
-BDD(InfluxDB) : Stockage des données timeseries - valeurs des capteurs avec timestamp, Optimiséé pour les données temporelles : insertion rapide et requêtes analytiques efficaces, Idéal pour suivre l'évolution des relevés en temps réel.
-Ngix : Serveur proxy pour gérer les requêtes vers l'API, Sécurise les communications avec HTTPS et bloque les requêtes malveillantes.
-Conteneur Docker : Hébergement les services - API, bdd, site web sur une machine virtuelle, Déployés sur un serveur cloud DigitalOcean pour assurer l'accessibilité, Sécurisés avec des certificats SSL et régulièrement mis à jour et Géré via Docker Compose pour simplifier le déploiement et la maintenance.
